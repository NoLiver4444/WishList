# `Main` (Основной контент)

Универсальный контейнер для отображения заголовка страницы, формы сортировки и
списка карточек. Поддерживает кастомный контент через `children` и динамическую
стилизацию через `variant`.

## Props

| Prop          | Тип                     | Default     | Описание                                                               |
|---------------|-------------------------|-------------|------------------------------------------------------------------------|
| `title`       | `string`                | `-`         | Заголовок страницы                                                     |
| `variant`     | `string`                | `'default'` | Модификатор стилей (добавляется как CSS-класс через `styles[variant]`) |
| `type`        | `string`                | `-`         | Тип сущности для передачи в CardList                                   |
| `sortOptions` | `Array<{label, value}>` | `[]`        | Массив опций сортировки                                                |
| `onAddClick`  | `() => void`            | `-`         | Обработчик клика по кнопке добавления                                  |
| `data`        | `Array<Object>`         | `[]`        | Данные для отображения в списке                                        |
| `children`    | `ReactNode`             | `-`         | Кастомный контент. Если передан, заменяет стандартный CardList         |

## Зависимости

- `react`: `useState` (отслеживание активной сортировки)
- `@/features/sort-cards`: Форма выбора сортировки
- `@/entities/ui/CardList`: Компонент списка карточек (рендерится по умолчанию)
- `./Main.module.css`: CSS-модули для стилизации

## Архитектура и поведение

- Инициализирует состояние `currentSort` первым элементом из `sortOptions` (если
  массив не пуст).
- Применяет динамические CSS-классы к корневому контейнеру на основе пропса
  `variant` через шаблонную строку.
- Условно рендерит `SortCardsForm` только при наличии переданных `sortOptions`.
- Реализует паттерн слотов (`children`): если `children` передан, рендерится он,
  иначе используется `CardList` с пропсами `type`, `data`, `onAddClick`.
- Позволяет гибко переиспользовать лейаут для разных разделов приложения без
  дублирования структуры.

## Пример использования

```jsx
<Main
  title="Мои желания"
  variant="wishes"
  type="wishes"
  sortOptions={[{ label: 'по дате', value: 'date' }]}
  data={wishlistItems}
  onAddClick={() => setModalOpen(true)}
/>
```
