# `AddCardModal` (Feature: add-card)

Модульная фича для создания сущностей через динамическую форму. Спроектирована по методологии **Feature-Sliced Design (
FSD)**. Генерирует поля на основе конфигурации, изолирует бизнес-логику в кастомном хуке и поддерживает системные хуки
для управления UI.

## Props

| Prop       | Тип                                    | Default | Описание                               |
|------------|----------------------------------------|---------|----------------------------------------|
| `isOpen`   | `boolean`                              | `false` | Флаг видимости модалки                 |
| `onClose`  | `() => void`                           | `-`     | Обработчик закрытия окна               |
| `onSubmit` | `(formData: Object) => void`           | `-`     | Колбэк успешной отправки формы         |
| `type`     | `'wishes' \| 'wishlists' \| 'friends'` | `-`     | Тип сущности (определяет конфиг полей) |
| `title`    | `string`                               | `-`     | Заголовок модального окна              |

## Структура модуля (FSD)

- **`ui/`**: Содержит React-компоненты.
    - `AddCardModal`: Основной оркестратор формы.
    - `FormField`: Атомарный компонент для отрисовки разных типов инпутов (number, date, select и др.).
- **`model/`**: Бизнес-логика.
    - `useAddCardForm`: Кастомный хук для управления состоянием, шагами числовых полей и обработкой событий.
- **`lib/`**: Вспомогательные функции.
    - `validate`: Чистая функция для проверки данных формы вне контекста React.
- **`config/`**: Описательная часть.
    - `fields.js`: Правила валидации, типы и метаданные полей.

## Архитектура и поведение

- **Децентрализация**: Логика валидации полностью отделена от отображения, что упрощает тестирование.
- **Умный ввод**:
    - Поля типа `number` поддерживают инкремент/декремент и фильтрацию нечисловых символов.
    - Поля типа `date` автоматически корректируются согласно `min/max` ограничениям при потере фокуса (`onBlur`).
- **Системные зависимости**: Использует общие хуки из `shared/hooks` (`useClickOutside`, `useEscClose`, `useEnterPress`)
  для соблюдения консистентности интерфейса.
- **Очистка**: При закрытии (onClose) состояние формы и ошибок сбрасывается до начальных значений (`emptyForm`).

## Пример использования

```jsx
import { AddCardModal } from '@/features/add-card';

// ...

<AddCardModal
	isOpen={isModalOpen}
	onClose={() => setModalOpen(false)}
	onSubmit={handleSave}
	type="wishes"
	title="Новое желание"
/>
```
