# `App` (Корневой компонент & Роутинг)

> Главный компонент приложения. Настраивает клиентскую маршрутизацию через
`react-router-dom`, импортирует глобальные виджеты (`Header`, `Footer`) и
> подключает базовые стили. Выступает точкой входа для SPA-роутинга.

## Маршруты (Routes)

| Путь         | Компонент (Ожидается) | Описание                       |
|--------------|-----------------------|--------------------------------|
| `/`          | `ProductsPage`        | Главная страница / Каталог     |
| `/wishlists` | `WishlistsPage`       | Управление вишлистами          |
| `/friends`   | `FriendsPage`         | Социальный раздел / Друзья     |
| `/calendar`  | `CalendarPage`        | Календарь событий              |
| `/settings`  | `SettingsPage`        | Настройки профиля и приложения |

## Зависимости

- `react-router-dom`: `Route`, `Routes` (декларативная маршрутизация)
- `@/pages/*`: Страницы приложения (используется Vite-алиас `@/`)
- `@/widgets/Header`, `@/widgets/Footer`: Глобальные виджеты layout
- `./styles`: Глобальные стили
- `Vite`: Обработка алиасов, HMR, статические импорты

## Архитектура & Поведение

- Компонент отвечает за декларативную настройку клиентских маршрутов.
- Стили подключаются единожды при инициализации, применяются глобально ко всем
  дочерним компонентам.
- Виджеты `Header` и `Footer` импортированы, но пока не отрендерены в JSX.
- Текущая реализация требует обёртки `<Routes>` и передачи компонентов в проп
  `element` для корректной работы роутера.
