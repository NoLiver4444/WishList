# `Header` (Шапка приложения)

Компонент верхней панели навигации и поиска. Управляет состоянием поискового
запроса, предоставляет ссылки на форму поиска и передаёт управляющие функции для
фокуса и очистки.

## Props

Компонент не принимает внешних пропсов. Использует внутреннее состояние для
управления поиском.

## Зависимости

- `react`: `useRef`, `useState` (управление фокусом и состоянием запроса)
- `@/shared/ui/Logo`: Логотип приложения (импортирован, зарезервирован для
  рендера)
- `@/shared/ui/Navigation`: Навигационное меню (импортирован, зарезервирован для
  рендера)
- `@/features/account-switch/AccountSwitcher`: Переключатель аккаунтов (
  импортирован, зарезервирован для рендера)
- `@/features/search-task/SearchForm`: Форма поиска (основной рендеримый
  элемент)
- `@/shared/ui/ShareButton`: Кнопка шаринга (импортирован, зарезервирован для
  рендера)
- `./Header.module.css`: CSS-модули для стилизации

## Архитектура и поведение

- Компонент хранит текстовый запрос в стейте `query` и управляет им через
  `setQuery`.
- Использует `useRef` для получения прямого доступа к DOM-элементам формы и
  инпута (`searchRef`, `inputRef`).
- `handleSearch`: Выполняет валидацию запроса (проверка на пустую строку после
  `.trim()`) и выводит результат в консоль. В текущей реализации служит
  заглушкой для будущей интеграции с API или роутером.
- `handleClose`: Очищает поле поиска и снимает фокус с инпута через `blur()`.
- В JSX рендерится только `<SearchForm>`, которому передаются стейт, рефы и
  обработчики. Остальные импортированные компоненты подготовлены для
  последующего добавления в лейаут шапки.

## Пример использования

```jsx
import Header from '@/widgets/Header';

const Layout = () => (
  <>
    <Header />
    <main>Контент страницы</main>
  </>
);
```
